Stateful behaviour

Some APIs on the sandbox are stateless - they return hard-coded happy path responses.

However, many APIs on the sandbox are stateful - they let you send test data through create (POST) or update (PUT) requests (or through an accompanying Test Support API) and then read back that same data through retrieval (GET) requests.

For stateful APIs where there is no way to clear down test data, the best way to start anew is to create a new test user.
